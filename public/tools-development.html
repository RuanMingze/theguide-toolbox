快速开始

核心概念

创建一个工具只需要两个文件，打包为 zip 格式即可导入到工具箱中使用。

1. 工具包结构

my-tool.zip ├── tool.json # 工具元数据 └── tool.tsx # 工具组件代码

2. 开发流程

  1. 创建 tool.json 文件，定义工具的元数据
  2. 创建 tool.tsx 文件，编写工具的功能代码
  3. 将两个文件打包为 ZIP 格式
  4. 在工具箱页面拖拽 ZIP 文件到导入区域
  5. 系统自动解析并添加工具到本地
提示

工具导入后会保存在浏览器的 localStorage 中，可以随时删除或导出分享。

tool.json 格式

基础结构

{ "name": "工具名称", "description": "工具简介，一句话说明这个工具是做什么的", "category": "工具类别", "icon": "Calculator", "author": "作者名", "version": "1.0.0", "tags": ["标签 1", "标签 2"] }

字段说明

字段 必填 类型 说明
name 是 字符串 工具名称（最多 20 字）
description 是 字符串 工具简介（最多 100 字）
category 是 字符串 工具类别（如：计算工具、转换工具、生成器等）
icon 否 字符串 图标名称，使用 Lucide React 图标，默认 Calculator
author 否 字符串 作者名
version 否 字符串 版本号，默认 1.0.0
tags 否 数组 标签数组，用于搜索
注意事项
  • 确保 JSON 格式正确，使用双引号
  • 必填字段不能缺失，否则导入会失败
  • 类别必须是预设类别之一，或自定义合理的类别名

打包步骤
  1. 将 tool.json 和 tool.tsx 放在同一目录
  2. 选中这两个文件，右键压缩为 ZIP 格式

常见问题

Q1: 导入工具失败

问题： 拖拽 zip 文件后显示"无效的工具包"

解决方案：

  • 检查 zip 文件结构，必须包含 tool.json 和 tool.tsx
  • 确保 tool.json 格式正确，包含必填字段
  • 确保 tool.tsx 导出默认函数
  • ZIP 文件打开后应该直接看到这两个文件，不要有子文件夹

Q2: 工具代码编译错误

问题： 导入后显示"代码编译失败"

解决方案：

  • 检查 TypeScript 语法是否正确
  • 确保使用了 React 18 的语法
  • 避免使用浏览器不支持的新特性
  • 检查是否使用了未定义的变量或函数

Q3: 样式冲突

问题： 工具样式与工具箱冲突

解决方案：

  • 使用 Tailwind CSS 的语义化颜色类
  • 避免使用内联样式
  • 使用 className 而不是 class
  • 不要修改全局样式

Q4: 工具无法保存结果

问题： 调用 onSaveResult 后没有反应

解决方案：

  • 确保传递的是可序列化的数据对象
  • 检查浏览器是否支持 localStorage
  • 确保数据大小不超过限制（1MB）

Q5: 如何调试工具

问题： 工具运行不符合预期

解决方案：

  • 打开浏览器开发者工具
  • 查看 Console 中的错误信息
  • 在代码中添加 console.log 调试
  • 使用 React Developer Tools 检查组件状态
